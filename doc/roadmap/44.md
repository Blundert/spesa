# Task 44 — Audit coerenza cache TanStack Query (v0.14.4)

## Problema

Con `staleTime: Infinity`, `invalidateQueries` marca una query stale ma non lancia il refetch se non ci sono subscriber attivi. Tutti gli schermi shell sono fratelli in TanStack Router: navigare tra di essi smonta lo schermo corrente. Risultato: dati stale visibili per breve tempo al ritorno su uno schermo (stale-while-revalidate).

Il bug era già stato corretto per `useMoveItemCategory` → `listItems` in v0.14.3. Questo task estende la correzione a tutte le mutazioni cross-screen rilevate dall'audit completo.

## Mutazioni corrette

### P1 — `src/hooks/useCatalogo.ts`

| Mutazione | Query | Fix |
|---|---|---|
| `useRenameCategory` | `qk.categories()` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |
| `useRenameItem` | `qk.listItems()` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |
| `useDeleteItemFromCatalog` | `qk.listItems()` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |

### P2 — `src/hooks/useListItems.ts`

Tutte le mutazioni chiamabili da SpesaScreen mentre ListaScreen è smontata:

| Mutazione | Fix |
|---|---|
| `useAddToList` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |
| `useAddNewItemToList` | `invalidateQueries(listItems)` → `refetchQueries({ type: 'all' })` |
| `useRemoveFromList` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |
| `useUpdateListQuantity` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |
| `useClearList` | `invalidateQueries` → `refetchQueries({ type: 'all' })` |

### P3 — `src/screens/SessioneScreen.tsx`

`deleteSession()` nel repository ricalcola i prezzi suggeriti degli articoli coinvolti (scrive su `db.items`), ma `handleDelete` non invalidava né `qk.items()` né `qk.purchasesForWeek(isoWeek)`.

Fix: aggiunta di entrambe le invalidazioni dopo `await deleteSession(sessionId)`.
