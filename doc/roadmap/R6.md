# Task R6 — Autocomplete lista della spesa (predefiniti + già usati)

**Data:** 2026-06-16
**Commit:** `feat: autocomplete on shopping list from base items and used items`

Feature aggiuntiva (post-rework). L'aggiunta voci alla lista ora ha un autocomplete alimentato
da un elenco base predefinito + gli articoli già usati.

## Implementato

### Nuovo `src/lib/baseItems.ts`
- `BASE_ITEMS: { name, category }[]` (~70 voci), con `category` allineata ai nomi delle categorie
  seed (Frutta e verdura, Frigo, Dispensa, Panetteria, Bevande, Altro) → le voci nuove finiscono
  nel gruppo giusto.

### `src/screens/ListaScreen.tsx`
- Aggiunto `useItems()`; costruito un **pool** deduplicato per `normalizeName`: prima i base,
  poi gli item del DB che sovrascrivono (mantengono la categoria reale). Ogni voce `{ name, categoryId }`.
- `matches` (input non vuoto): filtro per sottostringa normalizzata escludendo ciò che è già in
  lista; ordina prima chi inizia col testo, poi alfabetico; max 8.
- `quick` (input vuoto): chip rapide di voci non in lista — sostituiscono i 5 suggerimenti hardcoded.
- Tendina sotto l'input con le `matches` (nome + categoria) e riga "Aggiungi «testo»" se non è un
  match esatto. Tap/Invio → `handleAdd(name?, categoryId?)` che riusa `useAddNewItemToList` con la
  categoria del suggerimento (o dedotta dal pool), pulisce l'input e rimette il focus.

### Riuso
- `normalizeName` (`src/lib/normalize.ts`), `useItems` (`src/hooks/useItems.ts`),
  `useAddNewItemToList` (già con `categoryId` opzionale).
