## Area 2 — Export / Import JSON

## Area 3 — Sync git via GitHub Contents API

### Core (`src/lib/gitSync.ts`)
- **Config** in `localStorage`: `{ token, owner, repo, branch, path }` (default branch `main`,
  path `spesa-db.json`) + flag auto-sync. `isGitConfigured` = token+owner+repo+path presenti.
- Helper base64 UTF-8-safe (`TextEncoder`/`atob`), `encodePath` (encode per-segmento, supporta
  sottocartelle), `contentHash` (hash FNV-1a dei dati **escluso `exportedAt`** per rilevare modifiche reali).
- API GitHub (HTTPS, `Bearer` token, header `X-GitHub-Api-Version`): `getRemoteSha` (GET, 404→null),
  `putFile` (PUT base64 + sha se esiste = un commit), `getFile` (GET, fallback `download_url`).
- Alto livello: `syncPushForce()` (push manuale, sempre), `syncPushIfChanged()` (push solo se
  l'hash è cambiato — per l'auto), `syncPull()` (GET → parse → `isBackupData` → `importData`).
  Last-write-wins via `sha`.

### Auto-sync (`src/hooks/useAutoSync.ts`, montato in `AppShell`)
- `setInterval` ogni 3 minuti; il tick rilegge auto-flag + config da `localStorage` (così
  attivare/disattivare ha effetto dal tick successivo senza rimontare) e chiama `syncPushIfChanged`
  in modo silenzioso. L'effect non fa `setState` → nessun nuovo errore di lint.

### UI (`src/screens/ImpostazioniScreen.tsx`)
- Sezione **Sincronizzazione (GitHub)**: 5 input (token password, owner, repo, branch, path),
  "Salva configurazione", toggle **Sync automatico**, "Sincronizza ora" (push) e
  "Scarica da remoto" (pull con drawer di conferma che sovrascrive i dati locali). Stato `syncing`
  disabilita i pulsanti durante la chiamata; errori mostrati via toast.
