# Task R7 — i18n: infrastruttura react-i18next + traduzione completa

**Data:** 2026-06-16
**Commit:** `feat: internationalize the app (it/en) with react-i18next`

Parte 1/5 del blocco Impostazioni. Tutta l'interfaccia è ora traducibile IT/EN.

## Implementato

### Dipendenze
- `i18next`, `react-i18next` (persistenza lingua manuale via localStorage, niente language-detector).

### Infrastruttura (`src/i18n/`)
- `index.ts`: init i18next con `initReactI18next`, `resources { it, en }`, `lng` da
  `localStorage['app-language']` (fallback `it`), `interpolation.escapeValue:false`. Esporta
  `LANG_KEY`, tipo `Lang`, helper `categoryLabel(t, sortOrder)` per le 6 categorie seed.
- `locales/it.ts` + `locales/en.ts`: dizionari per area (`common`, `nav`, `budget`, `home`,
  `lista`, `spesa`, `storico`, `sessione`, `pasti`, `pianificazioni`, `supermercati`, `item`,
  `install`, `categories`). `en: typeof it` per tenerli allineati.
- `react-i18next.d.ts`: type augmentation → chiavi `t()` tipizzate.
- Importato come side-effect in `src/main.tsx`.

### Date localizzate (`src/lib/date.ts`)
- `DAY_NAMES_SHORT/FULL` e `MONTH_NAMES` ora mappe `{ it, en }`, selezione per `i18n.language`.
  Firme di `dayShort/dayFull/formatWeekLabel/formatShortDate/formatLongDate` invariate.

### Migrazione stringhe
- Tutte le schermate (`HomeScreen`, `ListaScreen`, `SpesaScreen`, `StoricoScreen`,
  `SessioneScreen`, `PastiScreen`, `PianificazioniScreen`, `SupermercatiScreen`,
  `ItemDetailScreen`) e i componenti (`AppShell`, `PriceKeypad`, `InstallPrompt`) usano
  `useTranslation()` + `t(...)`, toast inclusi (con interpolazione).
- Etichette categoria tradotte per `sortOrder` via `categoryLabel`; nomi articoli inseriti e
  chip base restano dati (IT).
- Default lingua: IT (il toggle arriva in R8).

## Verifica
- `tsc -b`: zero errori. `npm run build`: completato.
- `npm run lint`: zero errori NUOVI (restano i 2 preesistenti `set-state-in-effect`).
- Scan: nessuna stringa UI italiana hardcoded residua in screens/components.
