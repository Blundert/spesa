# Task 23 — Pasti: navigazione tra settimane

**Data:** 2026-06-16
**Commit:** `feat: navigate meal plan by week via search param`

Parte 3/5 della rework. La pianificazione pasti si scollega dalla settimana corrente: si
naviga avanti/indietro tra le settimane, ognuna col proprio piano (già conservato in DB).

## Implementato

### Helper data (`src/lib/date.ts`)
- `shiftISOWeek(isoWeek, delta)`: sposta una settimana ISO di ±N, calcolo interamente in UTC
  (niente scivolamenti di fuso), gestisce i confini d'anno. Costruito su `weekRange()`.

### Routing (`src/router.ts`)
- `pastiRoute` con `validateSearch` → `{ week: string }`, default `currentISOWeek()` se assente
  o non valido. Prima search-param tipizzato dell'app.

### Schermata (`src/screens/PastiScreen.tsx`)
- Settimana letta da `useSearch({ from: '/pasti' })` (non più costante di modulo).
- Selettore settimana nell'header: chevron ‹ / › + `formatWeekLabel(week)`; navigano con
  `navigate({ to: '/pasti', search: { week: shiftISOWeek(week, ±1) } })`.
- `useMealPlan(week)` / `useUpdateMealSlot(week)` ora seguono la settimana selezionata.
- Bottone reso onesto: "Vai alla lista della spesa" (naviga a `/lista`, la lista è globale).
  Rimosso il finto toast "Voci dai pasti aggiunte". I dish restano testo libero: la generazione
  automatica di ingredienti NON è in scope.

### Link a `/pasti` (search ora obbligatorio dal tipo)
- `HomeScreen.tsx` e `AppShell.tsx`: aggiunto `search={{ week: isoWeek }}` ai link verso `/pasti`.
