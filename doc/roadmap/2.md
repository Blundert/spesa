# Task 2 — Setup PWA

**Data:** 2026-06-15
**Commit:** `feat: setup PWA with vite-plugin-pwa and workbox`

## Cosa è stato fatto

- Installato `vite-plugin-pwa@^1.3.0` e `@vite-pwa/assets-generator`
- Creato `public/icon.svg`: icona carrello della spesa su sfondo `#2A2A2C`
- Generati automaticamente con `npx pwa-assets-generator --preset minimal`:
  - `public/pwa-64x64.png`, `pwa-192x192.png`, `pwa-512x512.png`
  - `public/maskable-icon-512x512.png` (per Android adaptive icons)
  - `public/apple-touch-icon-180x180.png` (per iOS home screen)
  - `public/favicon.ico`
- Configurato `VitePWA` in `vite.config.ts`:
  - `registerType: 'autoUpdate'` (aggiornamento silenzioso al reload)
  - Manifest completo: name, theme_color `#2A2A2C`, background `#F2F2F0`, display standalone
  - Workbox `generateSW`: precache tutti gli asset statici
  - RuntimeCaching CacheFirst per Google Fonts (TTL 1 anno)
- Aggiornato `index.html`:
  - `lang="it"`, `viewport-fit=cover` per safe area iPhone
  - Meta tag Apple per fullscreen/standalone
  - Link favicon.ico + icon.svg + apple-touch-icon

## Build output

- `dist/sw.js` + `dist/workbox-*.js` generati
- 21 entries precached (~301 KiB)

## Scelte prese

- `registerType: 'autoUpdate'`: per una PWA personal/single-user è il comportamento più semplice (nessun prompt "nuovo aggiornamento disponibile")
