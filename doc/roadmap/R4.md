# Task R4 — Storico pianificazioni

**Data:** 2026-06-16
**Commit:** `feat: meal planning history screen`

Parte 4/5 della rework. Schermata che elenca le settimane con almeno un pasto pianificato,
per sfogliare lo storico delle pianificazioni.

## Implementato

### Repository (`src/db/repositories/mealPlan.ts`)
- `getPlannedWeeks(): PlannedWeek[]` — legge tutte le righe `mealPlans`, riduce in JS a
  `{ isoWeek, mealCount }`, ordina dalla più recente (`localeCompare` desc, corretto su
  `"YYYY-Www"` zero-padded). Volumi piccoli (app personale) → nessuna query per settimana.

### Query key + hook
- `src/db/queryKeys.ts`: `plannedWeeks: () => ['plannedWeeks']`.
- `src/hooks/useMealPlan.ts`: `usePlannedWeeks()`; `useUpdateMealSlot`/`useClearMealPlan` ora
  invalidano anche `qk.plannedWeeks()` così lo storico resta aggiornato.

### Schermata (`src/screens/PianificazioniScreen.tsx`, nuova)
- Lista delle settimane pianificate: ogni card mostra `formatWeekLabel(isoWeek)` + n° pasti e
  linka a `/pasti?week=...` (riapre la settimana giusta). Empty state se nessuna settimana.
- Back → `/pasti` settimana corrente. Stile coerente con StoricoScreen.

### Routing + ingresso
- `src/router.ts`: route standalone `/pasti/storico` → `PianificazioniScreen`.
- `src/screens/PastiScreen.tsx`: link "Storico" nell'header verso `/pasti/storico`.
