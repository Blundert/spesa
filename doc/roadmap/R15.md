# Task R15 — Sync git bidirezionale + trigger (apertura/background/intervallo) + conflitti

**Data:** 2026-06-19
**Commit:** `feat: sync git bidirezionale con rilevamento conflitti e trigger apertura/background`

Evoluzione del sync git (R14): da "push cieco a intervallo" a **sync bidirezionale fast-forward**
con rilevamento conflitti, e tre trigger automatici. Nasce dalla domanda dell'utente: "cosa succede
se esco dall'app prima dei 3 minuti?" (il timer ripartiva da zero e le sessioni brevi non
sincronizzavano mai) e "git fetch + pull --rebase + push?".

Chiarito: non è git "vero" ma **un unico file JSON** (blob) via GitHub Contents API → niente
merge/rebase per-record; a livello di snapshot si fa pull / push / conflitto.

## gitSync.ts — riscrittura del livello sync
- Tracciamento dell'ultimo sync: `HASH_KEY` (hash dei dati) **+ nuovo `SHA_KEY`** (sha del file remoto).
- `getRemoteFile()` (GET → `{ sha, text }` o null su 404, fallback `download_url`); `putFile()` ora
  ritorna il **nuovo sha** dalla risposta. Rimossi `getRemoteSha`/`getFile`/`syncPushIfChanged`.
- **`autoSync({ allowPull })`** — confronta locale (hash) e remoto (sha) con l'ultimo sync:
  - solo locale cambiato → **push** (fast-forward, PUT con lo sha corrente);
  - solo remoto cambiato → **pull** (import, se `allowPull`);
  - entrambi cambiati → **`conflict`** (nessuna sovrascrittura automatica);
  - `allowPull:false` (background) = solo push, mai pull; se niente è cambiato localmente esce
    senza nemmeno fare la GET.
  - Ritorna `SyncOutcome` (`noop|pushed|pulled|conflict|notConfigured`).
- `syncPushForce()` (tieni locale: sovrascrive remoto) e `syncPull()` (tieni remoto) aggiornano
  entrambe le chiavi. Prima sincronizzazione senza baseline → trattata come conflitto (sicuro).

## syncRunner.ts (nuovo) — orchestrazione esito
- `runSync(qc, { announce })`: esegue `autoSync` e gestisce l'esito → `pulled` invalida le query +
  toast; `pushed`/`noop`/`notConfigured` con toast solo se `announce`; `conflict` mostra un toast
  persistente (`id: 'sync-conflict'`, `duration: Infinity`) con due azioni: **Tieni questo
  dispositivo** (`syncPushForce`) / **Scarica il remoto** (`syncPull` + invalidate).
- Guardia `inFlight` per evitare sync concorrenti tra i trigger.

## useAutoSync.ts — tre trigger (solo se auto attivo + configurato)
- **Apertura** (mount) → `runSync` bidirezionale (recupera modifiche da altri dispositivi).
- **Background/chiusura** (`visibilitychange` hidden + `pagehide`) → `autoSync({allowPull:false})`
  silenzioso, best-effort (su iOS non garantito se il sistema uccide la pagina).
- **Intervallo 3 min** mentre l'app è aperta → `runSync` (rete di sicurezza).
- Le condizioni si leggono dentro il trigger → il toggle ha effetto subito; l'effect non fa setState.

## Impostazioni
- "Sincronizza ora" ora chiama `runSync(qc, { announce:true })` (sync intelligente, non più push cieco).
- "Scarica da remoto" invariato (`syncPull` esplicito con drawer di conferma).
- Nuove chiavi i18n (it+en): `syncUpToDate`, `syncConflict`, `syncConflictKeepLocal`, `syncConflictPull`.
