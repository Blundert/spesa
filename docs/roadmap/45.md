# Task 45 — Fix categoryLabel per categorie custom (v0.14.5)

## Problema

Tutte le categorie con `sortOrder > 5` (Affettati=6, Congelatore=7, Cosmesi=9, Colazione=16, ecc.)
venivano mostrate come "ALTRO" nella Lista e nella Spesa. Le due sezioni "ALTRO" distinte nelle
screenshot corrispondevano a due categorie diverse (id=10 Cosmesi, id=17 Colazione) che ricevevano
entrambe il label sbagliato — non era un bug di cache.

## Root cause

`categoryLabel` in `src/i18n/index.ts` clampava ogni `sortOrder > 5` a `key = 5`:

```typescript
const key = sortOrder >= 0 && sortOrder <= 5 ? sortOrder : 5
return t(`categories.${key}` as 'categories.0')  // → sempre "Altro" per sortOrder >= 6
```

La funzione era stata scritta per le sole 6 categorie seed (sortOrder 0–5) e non aggiornata
quando l'app ha aggiunto il supporto alle categorie custom.

## Fix

Aggiunto parametro `name?: string` a `categoryLabel`: per `sortOrder <= 5` usa la traduzione,
per `sortOrder > 5` usa il nome DB passato dal chiamante.

Aggiornati i 3 siti chiamanti:
- `src/screens/ListaScreen.tsx` — catMap + group header render + fallback group usa `t('categories.5')`
- `src/screens/SpesaScreen.tsx` — catMap
