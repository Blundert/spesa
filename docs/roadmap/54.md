# Task 54 — Statistiche: selettore range di date (v0.21.0)

## Cosa è stato fatto

Aggiunto un selettore di range di date nella schermata Statistiche. Tutti i dati (KPI totale/sessioni/media/supermercato, grafico andamento, top articoli, ripartizione categorie) vengono filtrati al periodo selezionato.

### Range disponibili
- 7 giorni
- 30 giorni *(default)*
- 3 mesi
- 6 mesi
- Tutto (all-time)
- Giorni personalizzati (input numerico in BottomSheet)

### Comportamento
- La selezione persiste in `localStorage` (`statsRange`, `statsRangeCustomDays`)
- Il grafico andamento mostra le settimane dal `fromTs` alla settimana corrente (non più fisso a 12)
- Top articoli calcolati dagli acquisti nelle sessioni filtrate (non da `item.purchaseCount` all-time)
- `data-vaul-no-drag` sull'input numerico per evitare conflitti con il drag di Vaul

## Architettura
- Nuovo `src/lib/statsRange.ts`: tipi, getter/setter localStorage, `rangeToFromTs()`
- `getStats()` in `stats.ts`: nuovo parametro `fromTs: number | null`
- `useStats(fromTs)`: il parametro è ora esplicito, incluso nella query key
- `StatisticheScreen`: chip row + BottomSheet custom
