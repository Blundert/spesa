# Task 53 — Fix invalidazione cache statistiche dopo modifiche Catalogo (v0.20.6)

## Cosa è stato fatto

Tutte le mutazioni del Catalogo che modificano dati usati da `getStats()` ora invalidano `qk.stats()` in `onSuccess`, così le Statistiche si aggiornano in tempo reale senza reload.

Mutazioni corrette in `src/hooks/useCatalogo.ts`:
- `useRenameItem()` — aggiorna "Più acquistati" (nome articolo)
- `useDeleteItemFromCatalog()` — aggiorna "Più acquistati" e "Per categoria"
- `useMoveItemCategory()` — aggiorna "Per categoria"
- `useRenameCategory()` — aggiorna nome categoria in "Per categoria"
- `useDeleteCategory()` — aggiorna "Per categoria"

## File modificati
- `src/hooks/useCatalogo.ts`
- `src/data/changelog.ts`
- `package.json`
