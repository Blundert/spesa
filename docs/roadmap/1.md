## Scelte prese

- **Code-based routing** invece di file-based: più semplice da gestire ora, senza dipendenza
  dal plugin di generazione codice. Migrazione a file-based è sempre possibile in futuro.
- **Tailwind CSS v4** (non v3): la versione più recente usa il plugin Vite nativo,
  nessun file di configurazione separato. Funziona con `@import "tailwindcss"` diretto nel CSS.
- Il template Vite usa **React 19** e **TypeScript 6** — tutto compatibile con lo stack deciso.
